# AI Coding契约
+ 本文件规定了AI编程在执行代码变更时必须遵循的基本原则和规范。

## 1. 数据库变更规范
- 数据库需要有一个完整的初始化文件，这个文件在scripts/init-db.js中。
- 新增的数据库变更需要提供新增的SQL脚本文件来实现，脚本文件应放置在db_update_temp目录下。编号顺序递增。

## 2. 代码工程简洁性要求
- 除有额外明确要求外，每次开发时不需要将AI思考过程、Summary、Reasoning等内容写入代码注释或提交信息中。
- 代码应保持简洁、易读，遵循现有代码风格和规范。

## 3. 前端界面规范
- 增加任何元素时，请关注当前页面其他元素的高度尺寸信息，要和谐统一。
- 遵循现有的UI/UX设计规范，确保用户体验一致。

## 4. 文件存储规则
- 所有需要持久化的文件都应存放在uploadFiles目录下
- uploadFiles需要按业务功能再拆分目录
- 文件命名不要使用简单的md5或hash等无意义的字符，需要与功能和内容条目相关联，尽量见名知意
- 数据库中只需要存储这个文件的本地索引（使用相对目录，确保整个工程目录可以整体迁移）
- 一些临时的处理文件，放在tempFiles目录下

## deployScript
- 这个目录中存放所有部署脚本
- 我目前创建了startAll.sh，这个脚本不需要任何参数，执行后需要可以杀掉既有运行程序，然后前后端逻辑都完成启动，启动模式是后台运行，即远程SSH连接服务器开发时，SSH链接断开，也依然可以让服务继续工作
